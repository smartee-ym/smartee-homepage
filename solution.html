SpiderLive는 어떤것인가요?
SpiderLive에 관하여 자주받는 질문을 모아봤습니다!

1. SpiderLive란 무엇인가요?

SpiderLive는 스마트폰, 태블릿, PC 등의 멀티 디바이스에 최적화된 동영상 스트리밍 솔루션입니다. SpiderLive를 통해 모바일과 데스크탑에서 동영상 서비스를 제공할 수 있습니다.

2. 모바일에서만 동영상 서비스가 가능한가요?

PC에서도 가능합니다. PC의 경우 별도의 플레이어 개발이 필요합니다(SpiderLiveEnterprise 버전)

3. 동영상을 보기 위해 앱이 필요한가요?

SpiderLive는 다양한 플레이어를 지원합니다. 사용자가 모바일에서 동영상을 시청하기 위해 별도의 앱이 필요하지 않습니다.

4. 기존의 솔루션과 차이점이 무엇인가요?

기존의 솔루션은 너무 고가이거나, 외산 제품이어서 한글화 지원이 없고, 여러 업체를 거쳐 동영상 서비스를 구현 할 수 있습니다.
SpiderLive는 경제적인 가격으로 원스탑 서비스를 제공합니다.

5. 오픈 소스와의 차이점은 무엇인가요?

SpiderLive는 모든 상용 프로토콜을 지원하며, 지속적인 A/S 및 업데이트 서비스를 제공합니다.

6. 기존 솔루션에 비해 가격 경쟁력이 있나요?

TCO에 기반하여, Express 버전의 경우 약 20%의 가격 경쟁력을 가지며, Enterprise 버전의 경우 약 30%의 비용절감 효과를 가져 올 수 있습니다.

7. H/W를 포함한 서비스가 가능한가요?

가능합니다.

8. 라이선스 정책은 어떻게 되나요?

동영상 서버 1기를 기준으로 라이선스를 발급합니다.

9. 구축에 드는 기간은 얼마인가요?

SpiderLive는 멀티 플랫폼을 지원합니다. SpiderLiveExpress 버전은 바로 설치가 가능하며, SpiderLiveEnterprise 버전은 별도의 플레이어 개발 기간이 필요합니다.

10. 동영상 호스팅 서비스와의 차이점은 무엇인가요?

동영상 호스팅의 경우 자사의 귀중한 자산이 호스팅 업체에 존재하게 됩니다. 이는 자료 유출 등의 심각한 보안 문제를 야기할 수 있습니다.
또한 지원 프로토콜의 종류가 적어 모바일에서 동영상을 볼 수 없는 경우가 많습니다. 그리고 업로드 및 트래픽 용량에 제한이 있습니다.

11. SpiderLive의 사용을 위해 ActiveX 가 필요한가요?

ActiveX 가 필요 없습니다.

SpiderView는 어떤것인가요?
SpiderView에 관하여 자주받는 질문을 모아봤습니다!

1. SpiderView는 어떤 제품입니까?

SpiderView는 웹 애플리케이션의 Semantic Web Tag를 통해 다양한 디바이스 환경에 맞게 자동으로 레이아웃을 구성하며, 하나의 소스를 통해 다양한 디바이스 환경을 지원하므로, 개발 및 유지 보수를 용이하게 하는 멀티Screen, 반응형 웹 전문 솔루션입니다

2. 반응형 웹(Responsive Web Design) 이란 무슨 의미입니까?

반응형 웹(Responsive Web Design)은 HTML5 의 미디어 쿼리를 이용하여 하나의 소스로 제작된 컨텐츠가 데스크탑에 국한되지 않고 태블릿PC, 모바일 폰 등 다양한 크기의 디바이스 환경에 맞추어 해상도나 화면이 동적으로 변환되는 기법을 말합니다. 최근 다양한 모바일 기기가 보급되면서 하나의 사이트를 만들더라도 데스크탑/ 모바일 폰/ 태블릿pc등 서로 다른 해상도를 가진 디바이스들을 고려하지 않을 수 없게 됨으로써 N스크린의 시대로 불리는 다양한 디바이스의 해상도에 맞춰 사이트의 레이아웃을 변환하는 이슈에 맞춰 등장한 것이 반응형 웹입니다. 최근 구글에서도 모바일웹사이트 검색에 대한 엄격한 기준을 제시하고 반응형 웹디자인을 추천하고 있습니다. 모바일 웹사이트가 개발되지 않거나 제대로 서비스되지 않을 경우 모바일 검색에서 불리한 입장이 될 수밖에 없음을 명문화 하였습니다.

3. 어떤 상황의 기업들에게 SpiderView가 필요합니까?

웹 개발 관리에 있어 다음과 같은 과제를 안고 있는 기업이라면 SpiderView가 반드시 필요합니다.

  • • 모바일 웹으로 개발하려는데, 비용에 대한 이슈 발생
  • • 일반 웹과 모바일 웹 개발 유지보수에 대하여, 비용 문제 발생
  • • 일반 웹과 모바일 웹의 2가지 소스 관리에 대한 어려움
  • • 개발보다 유지보수에 더 많은 시간과 비용 투자
  • • 모바일 웹 신규 개발을 위한 레거시 시스템 분석의 난제
  • • 일반 웹과 모바일 웹을 동일한 URL 관리가 필요한 경우
4. SpiderView는 어떤 분야에 적용될 수 있습니까?

SpiderView는 다음과 같은 분야에 적용될 수 있습니다.

  • • PC웹과 모바일 웹 개발을 수행하는 프로젝트
  • • 기존 웹 사이트를 모바일 웹으로 서비스 하려는 경우
  • • 기타 멀티스크린 적용을 요하는 경우
5. SpiderView의 제품 구성은 어떻게 이루어져 있습니까?

SpiderView는 멀티스크린 지원을 위해 아래와 같이 독립적인 내부 모듈로 구성되어 있습니다.

  • • Layout Composer: 모바일 화면 레이아웃 구성을 제공해주는 모듈
    • - Semantic Web Tag Analyzer: HTML 내의 시멘틱 태그 분석 정보를 제공하는 모듈.
    • - Semantic Web Tag Support: HTML 내의 시멘틱 태그를 지원하지 않는 IE8에서 사용 가능하게 지원해 주는 모듈.
  • • Device UX module: 디바이스 기기에 맞는 UX를 제공해주는 모듈
    • - POHE: Plain Old HTML Element – 기존 HTML 태그 방식으로 쉽게 화면을 구성해주는 모듈
6. SpiderView를 설치하여 사용하기 위한 조건은 무엇입니까?

SpiderView 클라이언트를 위해서는 사용자 PC에서 다음과 같은 사항이 요구됩니다.

  • • 웹 브라우저: Internet Explorer 8 이상 (최신의 HTML5 지원 브라우저 권장)
    SpiderView 서버를 위해서는 다음과 같은 사항이 요구됩니다.
  • • JDK 1.6 이상
  • • 웹서버 J2EE 컨테이너: J2EE 1.6 이상
  • • 기 설치된 웹서버에 추가 설치됨.
  • • OS : Windows, Linux, Unix (Java 기반 기술 채택으로 플랫폼 독립적)
7. RWD (Responsive Web Design, 반응형 웹 디자인)과 SpiderView의 차이는 무엇입니까?

반응형 웹은 멀티스크린 지원을 위한 하나의 Spec, 개념이며, SpiderView는 멀티스크린 디자인 개념에 대하여, 개발자 입장에서 좀 더 쉽게 구현하기 위한 개발 프레임웍을 제공하는 제품입니다.
그리고 반응형 웹은 단지 화면 사이즈에 대한 체크를 통해, 화면을 구성해주는 개념인 반면, SpiderView는 디바이스 UX에 맞는 화면 구성 개념으로, 디바이스에 맞는 화면 제공이 가능한 장점을 가지고 있습니다

8. SpiderView가 멀티스크린 지원을 위해 해야 할 일은 무엇입니까?

SpiderView 적용을 위해 개발자는 HTML5 기반의 구조적 태그(시맨틱 웹 태그) 반영과 SpiderView전용 태그 작업이 필요합니다.

  • • HTML5 기반의 구조적 태그 : header, nav, section, article
    만일 이미 HTML5 기반으로 개발되어 있다면, 추가 작업은 필요 없습니다.
  • • SpiderView전용 공통 태그
9. SpiderView라이센스 정책은 무엇입니까?

SpiderView는 도메인 별 라이센스 방식으로 실행을 위해서 라이센스 키가 필요합니다.

10. SpiderView의 멀티스크린 기능으로 기존에 웹 상의 모든 기능이 모바일로 지원됩니까?

SpiderView는 기본적으로 모바일 기기에서 지원되지 않는 기술들은 멀티스크린 서비스에서도 제약 사항이 발생합니다. 예를 들면, 웹 플러그인 기반 기술은 모바일 기기에서 지원하지 않습니다 (ex: Flash, ActiveX, 등).

11. SpiderView와 모바일 앱(App) 방식과는 무엇이 다른가요?

SpiderView는 HTML5기반의 웹 기술입니다. 기존 Native App 방식과는 구현 방식이 전혀 다르게 되어 있습니다. 비유하자면, 기존 PC환경에서 C/S환경과 웹 환경으로 비교하셔도 됩니다.

12. SpiderView가 없다면, 멀티스크린 개발에 어떤 문제점이 있을까요?

SpiderView는 멀티스크린 기능 지원을 위한 각 종 웹 UI기술이 집약되어 있습니다. 만일 SpiderView가 없다면, 멀티스크린 기능을 위해 각 종 웹 기술을 별도로 구현해야 하는 어려움을 가지게 됩니다.

13. SpiderView와 기존 UX솔루션(예 : XPlatform)과의 차이점은 무엇인가요?

SpiderView는 HTML5기반의 웹 기술로 멀티스크린(N스크린)을 지원하는 제품입니다. 웹 환경에서 UI개발의 편의성을 위해 UI Component를 제공하는 UX솔루션과는 개념적으로 다릅니다.

14. SpiderView 도입시 어떤 사항을 준비 및 지원해 주어야 합니까?

SpiderView를 이용해 멀티스크린을 제공하기 위해서는 기존 HTML의 구조 분석이 중요합니다. 이를 토대로 SpiderView의 전용 태그 작업이 필요합니다.

15. SpiderView 시스템 운영 담당자가 별도로 필요합니까?

SpiderView는 개발 프레임웍 도구로서, 운영을 위한 담당자는 별도로 필요하지 않습니다.
